# YamieBot Session — April 9, 2026 (Session 1)

## Context
First session after the April 4 deep analysis. Focus: research current best-in-class tech stack and execute the first major architectural refactor — replacing the classical RAG pipeline with agentic RAG.

## Key Decisions Made

### 1. Full tech stack audit
Researched the April 2026 landscape via WebSearch:
- **LLM frontier**: Claude Opus 4.6, Claude Sonnet 4.6, GPT-5.4, Gemini 3.1 Pro all within 1-2 points on major benchmarks. Models trade wins by task type.
- **Classical RAG is dead**: ~75% of new enterprise RAG apps in 2026 use agentic or hybrid architectures. LlamaIndex itself published "RAG is dead, long live agentic retrieval."
- **Anthropic shipped Claude Managed Agents** on April 8, 2026 (day before this session). Public beta.
- **Forms clarified**: embedded Google Forms PDFs in Notion → migrating to Tally for n8n automation. Not our concern yet.

### 2. Framework/SDK decision
Chose **Anthropic Client SDK** (`anthropic==0.92.0`) with custom tool use for agentic RAG.

NOT the Claude Agent SDK (that's the Claude Code engine — designed for coding tasks, not chatbots).
NOT LangGraph (flexible but overkill, loses native MCP advantages).
NOT OpenAI (no longer top-tier for Dutch grounded answers; not where the user is invested).

**Model**: Claude Sonnet 4.6 as primary (near-Opus quality, significantly cheaper, fast enough for WhatsApp).

**Redis decision**: Kept. The Anthropic API is stateless — between WhatsApp webhook calls we need external storage for conversation history. Redis with TTL is the right tool regardless of what we used before. What changed: history is now passed as native Anthropic `messages` array instead of an injected string.

### 3. Architecture shift: classical → agentic RAG
Old flow: `embed → Pinecone (hardcoded 5 namespaces × 10) → filter → GPT-4o → answer`
New flow: `Claude decides → calls search_knowledge_base tool → Pinecone → results → Claude may search again → answer`

Key improvements:
- Claude controls retrieval (when, what, which namespace)
- Multi-search: Claude issues refined follow-up searches automatically
- No separate GPT-4o-mini question transformation step (completely removed)
- Conversation history as native Anthropic messages (not string injection)
- Greetings handled naturally — no tool call triggered
